# MASE 下一步优化方向

> 文档创建：2026-05-30
> 当前版本：MASE v0.4.0
> 当前最强成绩：LV-Eval EN 256k 88.71%（头条）/ 98.39%（历史峰值）；LongMemEval 80.2%（LLM-judge）

---

## 一、多模态记忆能力（优先级：高）

### 现状
MASE 目前仅支持文本记忆。所有写入记忆系统的内容均为纯文本，HybridRecall 三路召回也只针对文本向量和关键词。

### 目标
让 MASE 成为真正的多模态记忆引擎，支持以下内容类型的存储、索引和召回：

#### 1.1 图片记忆
- **写入**：用户上传图片时，调用视觉模型生成语义描述（caption）+ 结构化标签（场景、人物、物体、OCR 文字），写入 Entity Fact Sheet 和 Event Log
- **召回**：文字查询可以召回相关图片（"上次给我看的那个截图"）；图片查询可以找相似场景
- **存储**：图片以引用路径存储，不嵌入数据库，避免 SQLite 体积膨胀
- **实现参考**：LLaVA / Qwen-VL / 阿里通义千问多模态 API

#### 1.2 视频记忆
- **写入**：视频按关键帧采样，每帧生成描述，配合时间码和音频转录文字，构建视频记忆片段
- **召回**：支持"视频第几分钟讲了什么"这类时序查询
- **时序索引**：扩展 Event Log 的时间戳维度，记录视频内部的时间轴信息
- **实现参考**：Whisper 音频转录 + 视觉帧描述 + 时序 BM25

#### 1.3 表格记忆
- **写入**：识别 CSV / Excel / Markdown 表格结构，提取表头、行列关系，存入结构化 Fact Sheet
- **召回**：支持"上次那个预算表里第三季度的数字"这类精确查询；数值型字段支持范围查询
- **存储**：表格以 SQLite JSON 字段存储，FTS5 索引表头和单元格文本内容
- **实现参考**：pandas 解析 + 结构化 Fact Sheet 扩展

#### 1.4 文档记忆（PDF / Word）
- **写入**：长文档按章节切分，生成层级目录树（参考 PageIndex 的树状索引思路），每个节点存摘要
- **召回**：用 LLM 推理在目录树上导航（而非纯向量），精准定位相关章节
- **核心设计灵感**：PageIndex 的"similarity ≠ relevance，用推理代替向量搜索"思路可以移植到 MASE 的文档记忆层
- **差异化**：PageIndex 处理静态文档，MASE 处理动态记忆（文档内容会随用户修改而更新，需要 supersede 机制）

## 二、检索架构升级（优先级：高）

### 2.1 层级树状记忆索引
- **现状**：HybridRecall 是平铺召回，所有记忆条目在同一层级
- **目标**：建立记忆层级树——按时间段（本周/上月/去年）、主题（工作/生活/项目）自动分层
- **召回改进**：先用 LLM 推理确定"去哪层找"，再在该层做 HybridRecall，减少噪声
- **参考**：PageIndex 的树搜索 + MASE 现有 HybridRecall 组合

### 2.2 跨模态统一召回
- 文字查询可以同时召回文字记忆、图片记忆、表格记忆
- 统一相似度空间：各类型记忆都映射到同一向量空间（CLIP 或多模态 embedding）
- 召回结果按相关性混排，不再按类型分桶

### 2.3 记忆压缩与摘要
- 长期未访问的记忆自动压缩（多条相关记忆合并为摘要条目）
- 类似人类记忆的"遗忘曲线"：重要事实永久保留，细节随时间衰减
- 实现：定期 GC 任务 + 重要性评分（访问频率 × 最近度 × 用户标记）

## 三、安全与可靠性升级（优先级：中）

### 3.1 多模态 Prompt 注入防御
- 图片中可能嵌入不可见文字指令（白字白底、极小字体）
- 视频中可能有闪烁帧包含攻击指令
- 需要扩展 `untrusted-content.js` 类似的检测层到多模态输入

### 3.2 记忆隐私分级
- 对不同敏感级别的记忆设置访问控制（公开/私密/加密）
- 敏感字段（密码、银行卡号、身份证）自动 redaction 后存储

### 3.3 记忆完整性验证
- 扩展现有 tri-vault 的 git 审计能力到多模态内容
- 图片/视频内容的哈希校验，防止记忆被篡改

## 四、集成与生态扩展（优先级：中）

### 4.1 MCP 多模态工具
- 扩展现有 MCP Server，暴露图片/视频/表格记忆的读写接口
- Claude Desktop / Cursor 等客户端可以直接调用多模态记忆

### 4.2 LangChain / LlamaIndex 多模态适配
- 当前 BaseChatMemory 和 BaseMemory 适配层只支持文本
- 扩展为支持多模态消息（OpenAI 的 content array 格式）

### 4.3 流式写入
- 视频/长文档记忆写入支持流式处理，不阻塞主对话
- 后台异步建立多模态索引

## 五、基准测试扩展（优先级：低）

### 5.1 多模态记忆基准
- 当前 LV-Eval / LongMemEval / NoLiMa 均为纯文本基准
- 设计或寻找多模态记忆基准：图片-问答跨会话记忆测试

### 5.2 PageIndex 对比实验
- 在长文档问答场景上，对比 MASE 文档记忆层（树状索引+推理召回）vs PageIndex 原始方案
- 差异化：MASE 的动态更新（supersede）+ PageIndex 的推理检索 = 更强的文档记忆

## 六、技术债与工程优化（优先级：低）

- **WAL checkpoint 调度**：当前依赖 SQLite 自动 checkpoint，高并发写入时可能阻塞，考虑显式调度
- **HybridRecall 权重自适应**：当前 0.5/0.3/0.2 是固定权重，可以根据查询类型动态调整（时序查询时时间权重提高）
- **pre-commit hook 修复**：本地缺少 pre_commit 模块导致 commit hook 失败，需安装依赖
- **test 数量与 README 同步**：实际测试文件数量与 README 声称数字有出入，统一口径

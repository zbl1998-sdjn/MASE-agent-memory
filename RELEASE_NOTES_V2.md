# 📢 MASE 正式发布：开源 AI Coworker (告别向量检索的黑盒时代！)

**版本**：v2.0.0
**标签**：`Agent`, `LangGraph`, `SQLite`, `Anti-RAG`, `Memory-Augmented`

各位开发者，我们非常激动地宣布 **MASE (Memory-Augmented Smart Entity) 2.0** 正式上线！

这不仅是一次代码级别的重构，更是一次对当前 AI 智能体“长文本记忆”共识的**降维打击与理念颠覆**。

## 💡 我们为什么要彻底推翻旧版？
在 1.0 版本中，为了应对各种“记忆评测打榜”，MASE 堆砌了超过 600KB 的“上帝类（God Class）”、复杂的正则表达式和臃肿的 JSON 结构。更致命的是，我们发现：**在真实的长期工作流中，依赖传统向量数据库（Vector DB / RAG）来做记忆，是一条死胡同。**

当你今天告诉 AI “我改主意了，预算追加到两万”，向量检索很可能还会把一个月前“预算只有五千”的旧记忆一并召回，让大模型陷入时序错乱的幻觉中；当 AI 记错了你的关键信息，面对一堆高维的浮点数向量，你根本无从下手去物理修正它。

## 🔥 MASE：回归纯粹的白盒工业级架构

我们彻底抛弃了 RAG，转向了最古老但也最稳健的 **SQLite**，并使用 **LangGraph** 状态机重塑了整个智能体编排。

**MASE 带来了四个杀手级特性：**

### 1. 极致透明的“实体状态表”(Entity Fact Sheet) + FTS5
我们用 SQL 的 `Upsert` 机制完美解决了“时间冲突”问题。用户的最新偏好会瞬间覆盖旧数据，AI 在生成计划时，直接读取高纯度的最新状态表，再配合底层的 **FTS5 (BM25全文检索)** 抓取历史流水账（Append-Only）。告别“词汇孤岛”，毫秒级精准召回！

### 2. LangGraph 赋能的可视化工作流
彻底抛弃了过去的“黑盒大循环”，流程被拆解为：`路由决策` ➡️ `记忆读写(Notetaker)` ➡️ `任务规划(Planner)` ➡️ `外部动作(Action/MCP)` ➡️ `综合执行(Executor)`。每一步的状态变化清晰可见，调试效率提升十倍。

### 3. 记忆的绝对物理控制权 (CLI 面板)
AI 产生幻觉了？没关系。运行我们内置的 `mase_cli.py` 终端控制台，你可以像操作普通数据库一样，直接查看、添加或删除 AI 脑子里的某个具体事实。**你，才是系统的真正主宰。**

### 4. 异步“记忆垃圾回收” (Memory GC)
MASE 现在拥有一名后台守护者。`gc_agent.py` 会在闲时自动抓取最新的闲聊流水账，利用 LLM 将它们“提纯”为结构化的 JSON 事实，并无缝写入 SQLite。主界面 AI 永远只消费信噪比最高的知识精华！

### 5. MCP 外部工具能力
不仅仅是“聊天记录员”，而是真正的“数字同事”。通过原生的 MCP 扩展层，MASE 现在可以读取本地文件、获取系统时间，未来更将接入日历、邮件，并将动作结果写回记忆。

## 🛠️ 如何体验？

全新的 MASE 代码库已经清理完毕，所有的旧时代“屎山”都已被归档至 `legacy_archive/` 目录。

现在就拉取代码，在根目录下体验真正的白盒智能体：

1. **管理记忆**：`python mase_cli.py`
2. **异步提纯**：`python mase_tools/memory/gc_agent.py`
3. **全栈运行**：`python langgraph_orchestrator.py`
